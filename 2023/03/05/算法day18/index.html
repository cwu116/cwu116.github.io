树左下角的值

题目链接：https://leetcode.cn/problems/find-bottom-left-tree-value/

看讲解前思路：

层序遍历比较好解决这道题，层序遍历后求出每一层的节点值，返回最后一层也就是List中最后一个List的第一个值. 递归做法写在看题解后.

C#代码：

时间复杂度：O(N) 其中 n 为二叉树节点的个数.每个节点在递归中只被遍历一次.

空间复杂度：O(N) 队列最多存储N个节点.

看题解后的思路：

递归法需要找深度最大的叶子节点，每次遇见叶子节点，判断是否比当前存的最大深度大，如果大就覆盖返回值. 因为使用的是前序遍历，一定会先访问左边节点然后访问右边节点，当访问右边节点时，因为深度等于左边节点，不会覆盖.

C#代码：

时间复杂度：O(N) 其中 n 为二叉树节点的个数.每个节点在递归中只被遍历一次.

空间复杂度：O(height) 其中 height表示二叉树的高度，递归函数需要栈空间，而栈空间取决于递归的深度，因此空间复杂度等价于二叉树的高度.

从中序与后序遍历序列构造二叉树

题目链接:https://leetcode.cn/problems/construct-binary-tree-from-inorder-and-postorder-traversal/

看题解前的思路:

自己想没有想出来，思路写在看题解后.

C#代码：

时间复杂度：O(N) 其中 n 为二叉树节点的个数.每个节点在递归中只被遍历一次.

空间复杂度：O(height) 其中 height表示二叉树的高度，递归函数需要栈空间，而栈空间取决于递归的深度，因此空间复杂度等价于二叉树的高度.

看题解后的思路：

通过中序遍历和后序遍历的性质，可以去拆分数组.

  1. 后序遍历的最后一个是根节点.
  2. 通过这个根节点在中序遍历的位置，切割中序遍历的数组成左右两边
  3. 根据中序遍历左边的长度，切割后序遍历的数组成左右两边
  4. 递归处理左右区间

需要注意中序数组大小和后序数组大小相同，也就是说切割后的长度也应该相同.

同时也需要注意切割的标准，在递归的时候需要保持一致.

还有每次新开空间给数组很占内存，和耗时，可以传入切割的index达到一样的效果.
