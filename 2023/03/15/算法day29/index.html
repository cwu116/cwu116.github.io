算法day29-递增子序列,全排列,全排列II

Created2023-03-15|Updated2023-03-15|算法
|Post View:

递增子序列

题目链接：https://leetcode.cn/problems/non-decreasing-subsequences/

看题解前思路：

做的时候以为和昨天子集II区别没有太大，想添加一些条件实现这题，但是是有区别的，子集II可以排序但是这题不能对原题排序，所以不能在子集II题解的基础上解决该题. 思路写在题解后

c#代码：

时间复杂度：O(N* 2^N) n个位置每次考虑选或者不选和加入答案的时间代价.

空间复杂度：O(N) 递归的层数，最多递归N层.

看题解后的思路：

因为不能使用排序，所以不能用子集II的方法去重. 做法为使用一个哈希表记录在这一层里该元素是否被使用，如果被使用过了直接continue进入下一个循环. 并且判断比上一个返回列表的数大判断是否递增.

全排列

题目链接：https://leetcode.cn/problems/permutations/

看题解前思路：

第一次做排列一开始写暂时没有思路，只知道和组合不一样，组合不同顺序只算作为一种，排列不同顺序算多种.

c#代码：

时间复杂度：O(N* N！) backTrack调用次数为O(N!)和对于每个节点把结果添加进返回数组O(N)的时间

空间复杂度：O(N) 除答案数组外，空间复杂度取决于递归的栈深度，最大为N层.

看题解后的思路：

因为是求排列，不是求组合，就不用使用startindex避免重复了，因为每次都要从头搜索. 但是需要使用一个used数组，去标记已经选择的元素. 叶子节点就为返回值的地方，当存储列表长度等于数组长度时，说明找到了排列.

全排列II

题目链接：https://leetcode.cn/problems/permutations-ii/

看题解前思路：

在上一题的基础上，因为数组中多了重复数字，所以要做一个去重的操作. 因为没有了startindex帮助去重，这里判断了上一个数是否被使用，如果被使用且这个数和上一个数相等，则在这层跳过.

c#代码：

时间复杂度：O(N* N！) backTrack调用次数为O(N!)和对于每个节点把结果添加进返回数组O(N)的时间

空间复杂度：O(N) 除答案数组外，空间复杂度取决于递归的栈深度，最大为N层.

看题解后的思路：

去重一定要对元素进行排序，这样才方便通过相邻节点来判断是否重复使用了.

这里有个地方需要注意，used[i-1] == true 和 used[i-1] == false都可以运行成功.

区别在于used[i-1] == false对树层去重，used[i-1] == true对树枝去重.

nums[i-1] == nums[i] 且 used[i-1] == true说明在递归中，在上一层或者更上面被使用过了.

nums[i-1] == nums[i] 且 used[i-1] == false说明在同一层，已经使用过并且已经回溯了，所以是false.

树层去重效率更高.
