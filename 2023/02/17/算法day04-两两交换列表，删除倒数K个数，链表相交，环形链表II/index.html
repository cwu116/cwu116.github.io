看题解后的思路:

看了题解恍然大悟，之前刷过一次缺忘记了这个双指针的方法. 本质上是一个数学问题. 快的指针先走n + 1步，然后同时和慢的指针再一起走.

这里先忽略那个1去理解n(倒数的步数)，走了n步 还剩 k-n 步，这个步数就是正着走时要求的步数.

这个时候快指针再和慢指针一起出发，快指针结束，慢指针到达指定位置. +1是为了少走一步方便删除.

链表相交

题目链接：https://leetcode.cn/problems/intersection-of-two-linked-lists-lcci/

看讲解前思路：

之前刷过一次记得思路，本质上也是数学问题.

假设链表A为 1,2,3 —- 4,5

假设链表B为 1,1 —- 4,5

链表A遍历完一次需要循环5次，

链表B遍历完一次需要循环4次，

可以发现，A遍历完后，接着遍历B: 1,2,3,4,5,1,1

和B遍历完后，接着遍历A: 1,1,4,5,1,2,3 会在交点处相遇.

环形链表II

题目链接：https://leetcode.cn/problems/linked-list-cycle-ii/

看讲解前思路：

刷过一次只记得用快慢指针，忘记了数学推导，一开始自己没有想出来.

C#代码：
看题解后的思路:

借卡哥的图用一下，可以把图画出来拆分一下走过的路径.

慢指针走的路程为 x + y

快指针走的路程为 x + y + n(y + z) n(y + z)为快指针多走的圈数

快指针是慢指针两倍速 2x + 2y = x + y + n(y + z)

等价于 x + y = n(y + z)

x = n(y + z) - y

假设n = 1, x = z

最好情况下 慢指针和从起点新建一个指针一起走，慢指针走z步，新指针走x步就可以相遇.
