哈希表

常见的哈希结构：

数组

Hash Set: 存储键

Hash Map：存储键值对

使用哈希表可以快速判断元素是否在集合里，遍历数组花费O(n)的时间，哈希表O(1)时间就可以做到

当需要快速判断元素是否在集合里时，考虑使用哈希法.

有效的字母异位词

题目链接：https://leetcode.cn/problems/valid-anagram/

看讲解前思路：

使用字典(HashMap)存储，Key存储字符，Value存储个数. 先遍历第一个字符串，把每个单词对应数量记载到map中，然后遍历第二个字符串，减去第二个字符串中的元素个数. 如果发现有字典中没有的，直接返回false. 最后遍历字典判断每个字符的个数是否为0. **有点忘记c#字典语法了**，面试前需多看.

时间复杂度：O(N)

空间复杂度：O(N)

看题解后的思路：

想复杂了，数组就是最简单的哈希表，使用小写字母的数量长度的数组存储字母出现的次数. s[i] - ‘a’可以表示这个数相对于小写字母a来说的位置

两个数组的交集

题目链接：https://leetcode.cn/problems/intersection-of-two-arrays/

看讲解前思路：

和上一题思路类似，先把第一个数组的数放进hashSet，然后遍历第二个数组，如果遇见有hashSet中的数的话，加进返回的数组中. 同时删除这个数避免数组二中有重复的该数.

C#代码：

时间复杂度：O(M+N)

空间复杂度：O(M+N)

看题解后的思路:

set 占用空间比数组大，而且速度要比数组慢，set把数值映射到key上都要做hash计算，不一定要无脑用哈希表

两数之和

题目链接：https://leetcode.cn/problems/two-sum/

看讲解前思路：

最容易想到的就是暴力算法. 时间复杂度为O(N^2). 在这里我们使用哈希表. 巧妙的地方在于存储数值在Key，存储index在value上. 因为哈希表不能通过value找key，如果反着存的话我们就找不到对应index了. 思路为遍历数组，如果hashMap里有另一个和现在的数可以组成目标数，返回. 不能的话就把现在的数加进hashMap里.

在这里c#字典会不能添加重复的key，也就是哈希冲突. 所以这里用Java语言写答案. 还有为什么重复的数可以覆盖的问题，重复的数有两种情况：1.两个数可以组成目标数，在第二个数添加之前，就已经判断了加起来是否可以组成，返回结果. 2.两个数不能组成目标数，第二个数覆盖了第一个数，对于结果来说没有影响.

看题解后的思路:

当查询元素是否出现过，或者元素是否在集合中，考虑使用哈希法. 需要了解什么情况需要使用数组还是hashSet还是hashMap. 这道题因为需要存储元素对应的下标，所以使用hashMap
