算法day01-二分查找,删除指定数字

Created2023-02-14|Updated2023-02-15|算法
|Post View:

开始系统性刷题，第一天是很久都没刷过的二分查找和-删除指定数字，在这里先po一下自己没看题解的思路

二分查找

C#代码

由于很久没刷过二分查找，有点忘记了，依稀记得二分查找有细节需要注意，但是忘了具体是什么地方，应该是在left <= right 的处理上
在看完卡哥题解后继续分析.二分的整体思路是通过判断中位数缩小范围，如果中位数大于目标数，右指针挪到中位数index-1的位置,反之小于目标书，左指针挪到中位数index+1的位置

第一步：初始化左右指针
第二步：建立While循环 left <= right 时执行
第三步：求出mid 中位数的 index
第四步：判断中位数和target的区别然后做不同操作

时间复杂度:O(LogN)

为什么是LogN的原因： 因为每次缩短数组长度的一半N/2，如果执行x次， 也就是N/2^x = 1, 可以推出 x = logN

空间复杂度: O(1)

看完题解之后的分析：

题目的前提是有序数组，无序数组不能使用二分法，在看到是有序数组时考虑是否能使用二分法
边界条件(left <= right) 还是(left < right)
right = mid 还是 right = mid -1

分为两种写法：一种为左闭右闭[ left,right ]，或者左闭右开[ left,right )

第一种： 左闭右闭[ left,right ]
使用 while(left <= right), 因为left == right 有意义
right = mid -1, 因为nums[ mid ] 一定不是target，下一个需要查找的就是mid-1的位置
初始化 left = 0, right = lenght -1

第二种:或者左闭右开[ left,right )
使用 while(left < right), 因为left == right 没有意义
right = mid, 因为下一次也不会比较右边界(因为是左闭右开)
初始化 left = 0, right = lenght

需要注意一下mid中位数的求值方法：防止溢出
int middle = left + ((right - left)/2)

移除元素

C#代码

在卡哥的提示下了解了用双指针做这道题，思路为左指针为需要目标值时，替换右指针的值到左指针，被替换后右指针像左挪动
在这里可以发现右指针一定为目标数，所以right –， 但是左指针替换后仍然可能为目标数，需要到下一次循环再做判断

时间复杂度：O(N)

空间复杂度: O(1)

看完题解之后的分析：

在力扣上看见一个题解也许更方便
拷贝覆盖：
主要思路为用一个index记录当前非目标值的下标, 遇见了非目标值时, 给nums[ index ]的值为当前值, index++
遇见目标值时不做操作
这种思路在移除元素较多时更适合使用，最极端的情况是全部元素都需要移除，遍历一遍结束即可
之前写的思路在移除元素较少时更适合使用，最极端的情况是没有元素需要移除，遍历一遍结束即可
